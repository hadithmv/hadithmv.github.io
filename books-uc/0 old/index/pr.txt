Of course. I can certainly help you redesign your homepage to be more modern, efficient, and easier to manage. Your idea of using a dynamically loaded JSON object is excellent and is a standard practice in modern web development for exactly the reasons you mentioned: it separates your content from your presentation, making the site much easier to update and scale.

Here is a complete, ready-to-use redesign of your homepage, incorporating a data-driven approach with a JSON object, dynamic rendering, and a live search functionality.

### Key Improvements in This Redesign:

*   **Data-Driven Content:** All the links and books are stored in a single `siteData` object in the JavaScript file. To add, remove, or change a book, you only need to edit this object, not the HTML file.
*   **Dynamic Rendering:** The JavaScript code automatically generates the HTML for each section and book from the `siteData` object. This eliminates hundreds of lines of repetitive HTML.
*   **Live Search Bar:** A search bar is added at the top. It filters the content in real-time as the user types, searching across Arabic, Dhivehi, and English titles, as well as keywords.
*   **Modern & Clean Design:** The new CSS provides a cleaner, more modern aesthetic with a consistent design, improved readability, and subtle animations. It is also fully responsive.
*   **Simplified HTML:** The `index.html` file is now incredibly simple and clean, serving as a skeleton that gets populated by the JavaScript.

### How to Use the New Design:

1.  **Save the files:** Create a new folder for your redesigned site. Save the HTML as `index.html`, the CSS as `style.css`, and the JavaScript as `script.js` inside that folder.
2.  **Update URLs:** In the `script.js` file, inside the `siteData` object, make sure the `url` values for each item point to the correct path for your book files (e.g., `../books/quranHadithmv.html`). I have kept the ones from your original file.
3.  **Manage Content:**
    *   **To add a new book:** Simply add a new object to the `items` array of the relevant section in `siteData`. For example: `{ titleAr: "...", titleDv: "...", url: "..." }`.
    *   **To add keywords for searching:** Add a `keywords` property to any item, like this: `{ ..., keywords: "keyword1 keyword2" }`.
    *   **To add a status:** Add a `status` property, like this: `{ ..., status: "(In Progress)" }`.
4.  **Open `index.html`:** Open the `index.html` file in your browser to see the new homepage.
